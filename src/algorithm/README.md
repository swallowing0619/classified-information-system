#功能一的加密算法

第一个功能的实现，即大文件加密传输，需要用到的算法包括MD5文件摘要算法、RC4流密码算法、RSA非对称加密算法。

##实现的逻辑如下：
加密 
A选择本机的一个文件file后，自动生成一对公私钥：将公钥Y1发送给对方B；私钥Y2作为自己的加密密钥，把输入的RC4加密的密钥key和文件经过摘要算法生成的哈希值HASH1连在一起加密，
得到密文后发送给B，再利用key加密file生成已加密的字节流传输给B。

解密
B接收A的公钥后，将第一段密文用公钥Y1解密,可得到文件加密的密钥key和文件哈希值HASH1，再用key解密接收到的文件字节流，还原文件后，利用摘要算法生成接收并解密完的文件哈希值HASH2,
对比HASH1与HASH2，若相同，则传输的文件是完整的。打开接收并解密的文件，若与传输前的文件一致，则解密成功。
